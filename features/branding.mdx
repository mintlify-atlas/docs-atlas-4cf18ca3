The branding page at `acmcsuf.com/branding` is the central resource for ACM at CSUF brand assets. It displays logo files and brand colours for every team, with SVG and PNG download links.

## Page structure

Source: `src/routes/(site)/branding/+page.svelte`

The page imports the `Branding_Details` array from `data.ts` and renders a `<TeamCard>` (`branding-card.svelte`) for each entry. A header section shows the Boardify capybara illustration (`/assets/figma-capy.svg`) alongside the page title and description.

The page footer links to:
- `/global.css` — the global stylesheet, useful for colour variable reference.

## Data structure

Defined in `src/lib/public/branding/branding.ts`:

```ts
export interface Branding {
  title: string;        // team display name
  description: string;  // team description
  teamLogo: TeamLogo;
  teamColor: TeamColor;
}

export interface TeamLogo {
  image: string;  // path to image for display
  alt: string;    // alt text
  title: string;  // human-readable logo title
  svg: string;    // path to SVG asset for download
  png: string;    // path to PNG asset for download
}

export interface TeamColor {
  name: string;   // colour name, e.g. "Primary Blue"
  hex: string;    // hex value, e.g. "#06a3ff"
  rgb: string;    // RGB values, e.g. "6, 163, 255"
}
```

## Adding or updating branding

<Steps>
  <Step title="Add asset files">
    Place the new or updated SVG and PNG logo files in the `static/assets/` directory.
  </Step>
  <Step title="Update data.ts">
    Open `src/routes/(site)/branding/data.ts` and update the relevant entry — or append a new `Branding` object for a new team — with the correct asset paths and colour values.
  </Step>
  <Step title="Update the type if needed">
    If you are adding a new field to the `Branding`, `TeamLogo`, or `TeamColor` interfaces, update `src/lib/public/branding/branding.ts` and the `branding-card.svelte` component accordingly.
  </Step>
  <Step title="Submit a pull request">
    Open a PR against `main`. Changes go live after merge and deployment.
  </Step>
</Steps>
