acmcsuf.com uses [SvelteKit's file-based routing](https://kit.svelte.dev/docs/routing). Every file with a `+` prefix inside `src/routes/` becomes an endpoint. All site pages live inside the `(site)` route group, which acts as a layout wrapper without affecting the URL.

## File conventions

| File | Purpose |
|---|---|
| `+page.svelte` | The UI component rendered for that route |
| `+page.ts` | Runs on server and client to load data before the page renders |
| `+layout.svelte` | Wraps all child routes with shared UI (nav, footer) |
| `+error.svelte` | Displayed when a page throws an error |

<Note>
The root `+layout.svelte` at `src/routes/(site)/+layout.svelte` is rendered around every page on the site. It includes the top navigation bar and footer.
</Note>

## Dynamic routes

SvelteKit uses square-bracket syntax for dynamic segments:

| Pattern | Example URL | Matched value |
|---|---|---|
| `[id]` | `/blog/42` | `id = "42"` |
| `[team]` | `/workshops/dev` | `team = "dev"` |
| `[season_id]` | `/lc-dailies/2024-spring` | `season_id = "2024-spring"` |
| `[username]` | `/@ethanthatonekid` | `username = "ethanthatonekid"` |
| `[repo_name]` | `/@ethanthatonekid/acmcsuf.com` | `repo_name = "acmcsuf.com"` |

<Note>
The `[x+40]` prefix in `[x+40][username]` is SvelteKit's encoding for the `@` character (Unicode code point 40 in hex). This maps the `/@username` URL pattern.
</Note>

## Data loading with `+page.ts`

Routes that need server data define a `load` function in a `+page.ts` file:

```typescript src/routes/(site)/events/+page.ts
import type { PageLoad } from './$types';

export const load: PageLoad = async ({ fetch }) => {
  const response = await fetch('/events/upcoming.json');
  const events = await response.json();
  return { events };
};
```

The returned object is passed as `data` props to the corresponding `+page.svelte`.

## Server hooks

Before any route renders, `src/hooks.server.ts` intercepts every request. It runs two handlers in sequence:

1. **`shortener()`** — checks if the URL matches a shortlink in `links.json` and redirects if so
2. **`handleTheme`** — reads the `theme` cookie and injects `data-theme` into the HTML for light/dark mode

```typescript src/hooks.server.ts
import { sequence } from '@sveltejs/kit/hooks';
import { shortener } from '$lib/server/links/shortener';

export const handle = sequence(shortener(), handleTheme);
```
